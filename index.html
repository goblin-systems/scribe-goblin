Ranking

Hybrid search picks one weight pair based on query shape: short, medium, or long. For any single search, 5 parameters are active: keyword weight, semantic weight, recency max, RRF k, and the selected short/medium/long bucket.

Formula used per result: `kw_rrf = keyword_weight / (rrf_k + keyword_rank)` and `sem_rrf = semantic_weight / (rrf_k + semantic_rank)`, then `fused = kw_rrf + sem_rrf + recency_boost`. Only one bucket pair (short, medium, or long) participates per search. Semantic results below the relevance threshold are discarded before ranking, so recent but irrelevant semantic matches do not survive just because they are new. These values rerank live while you edit them.
